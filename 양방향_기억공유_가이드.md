## 🎯 양방향 기억공유란?

### Claude ↔️ 개발자 간 정보 동기화
```
첫 번째 세션:
개발자 → Claude (깃허브 URL, 요구사항)
         ↓
Claude: CLAUDE.md, 양방향_기억공유_가이드.md 생성
         ↓
개발자 ← Claude (가이드 문서)

이후 세션:
개발자: "CLAUDE.md 읽어줘"
         ↓
Claude: CLAUDE.md 자동 참고 → 정확한 작업 진행
```

### 이점
✅ **연속성**: 매번 상황 재설명 불필요
✅ **정확성**: 이전 분석과 결정사항 반영
✅ **효율성**: 작업 시간 단축
✅ **추적성**: 모든 작업 이력 기록

## 🤖 Claude Code 사용 규칙

### Rule 1: 항상 CLAUDE.md부터 읽기

**작업 시작 전**:
```
"현재 프로젝트 상태를 확인하고 싶어요"

Claude의 반응:
→ CLAUDE.md를 자동으로 참고
→ 현재 상황 요약
→ 우선순위 제시
```

### Rule 2: 작업 요청 형식 (명확하게)

#### 좋은 예시 ✅
```
"petcare-plus-v2의 이메일 발송 기능을 고쳐줘.

상황:
- 상담 신청 폼 제출 후 이메일이 안 옴
- RESEND_API_KEY가 미설정 상태

요청:
1. api-server.js에 이메일 에러 처리 강화
2. App.jsx에서 사용자 메시지 명확화
3. .env.local 설정 가이드 제공
4. 로컬 테스트 방법 설명

우선순위: 최우선
배포 영향: 이 기능 없이 배포 불가
"
```

#### 나쁜 예시 ❌
```
"이메일 고쳐줘"
(너무 모호함 - Claude가 상황을 다시 물어봐야 함)

"모든 기능을 다 고쳐줘"
(너무 광범위 - 우선순위 불명확)
```

### Rule 3: 작업 결과 형식

**Claude가 제공할 정보**:
```
1. 변경사항 요약
2. 수정된 파일 목록
3. 테스트 방법
4. 배포 영향도
5. 다음 단계
6. CLAUDE.md 업데이트 제안
```

### Rule 4: 지속적 추적

**모든 작업 후에**:
```
Git commit 메시지:
"[작업타입] 간단한 설명

세부사항:
- 수정한 파일들
- 테스트 결과
- 남은 이슈

https://claude.ai/code/session_[ID]"
```

## 📊 작업 흐름 예시

### Scenario 1: 이메일 발송 기능 고치기

#### Step 1: 상황 파악
```
사용자: "이메일 발송이 안 돼. 뭐가 문제야?"

Claude:
1. CLAUDE.md의 "1️⃣ 이메일 발송 미작동" 섹션 참고
2. 원인 분석:
   - RESEND_API_KEY 미설정
   - 환경변수 설정 필요
3. 해결책 제시:
   - https://resend.com에서 API 키 발급
   - .env.local 설정
   - 테스트 방법
```

#### Step 2: 코드 수정
```
Claude:
1. api-server.js 강화 (에러 처리)
2. App.jsx 수정 (사용자 메시지 명확화)
3. api/send-email.js 검증 강화
```

#### Step 3: 테스트
```
사용자: "로컬에서 테스트 해봤는데 됨!"

Claude:
1. Vercel 배포 환경변수 설정 가이드
2. 배포 후 재테스트 방법
3. 문제 발생 시 대응책
```

#### Step 4: CLAUDE.md 업데이트
```
Claude:
```
✅ 1️⃣ 이메일 발송 미작동 → FIXED
- 날짜: 2026-02-28
- 커밋: abc123def
- 남은 이슈: Vercel 배포 후 테스트 필요
```

### Scenario 2: 배포 전 최종 점검

```
사용자: "배포 준비됐나?"

Claude:
1. CLAUDE.md에서 "✅ 테스트 체크리스트" 확인
2. 체크되지 않은 항목 나열
3. 부족한 부분 수정 요청
4. 배포 준비 완료 확인 후 가이드 제공
```

## 🔄 정보 동기화 방법

### Claude → GitHub (기억 저장)

**매 작업 후**:
```
1. 작업 내용을 CLAUDE.md에 반영

```
사용자: "안녕, CLAUDE.md 읽어주고 현황 보고해줄래?"

1. GitHub에서 CLAUDE.md 자동 읽기
2. 현재 상황 분석
3. 진행 상황 요약
4. 다음 할 일 제시
5. 필요한 정보 수집
```

Claude: (코드 수정 시작)
"✅ api-server.js 수정 완료
✅ api/send-email.js 수정 완료
✅ App.jsx 수정 완료

테스트 방법:
1. npm run dev:full 실행
2. 상담 신청 폼 제출
3. PETCARE_ADMIN_EMAIL로 이메일 수신 확인

다음 단계:
→ Vercel 배포 후 재테스트
→ CLAUDE.md 업데이트"
```
