# 🔄 Claude.ai ↔️ Claude Code 양방향 기억 공유 시스템

## 목표
Claude.ai와 Claude Code가 서로 기억을 공유하고 보완하여, 사용자가 수정/보완 작업을 반복하지 않도록 함.

---

## 📋 핵심 원칙

### 1. **단일 진실의 원천 (Single Source of Truth)**
```
CLAUDE.md = 프로젝트의 모든 지식

- 현재 상태
- 완료된 작업
- 미완료 작업
- 중요 교훈
- 환경 설정
- 작업 규칙
```

### 2. **자동 업데이트 규칙**
```
작업 완료 시마다:
→ CLAUDE.md 업데이트
→ GitHub 커밋
→ 양쪽 Claude 모두 최신 정보 확보
```

### 3. **실수 방지 체크리스트**
```
모든 작업 전:
1. CLAUDE.md 읽기
2. 현재 상태 확인
3. 작업 계획 수립
4. 실행
5. CLAUDE.md 업데이트
```

## 🔧 트러블슈팅

### **Q: Claude Code가 CLAUDE.md를 안 읽어요**
```
A: 
1. 파일 위치 확인 (프로젝트 루트에 있어야 함)
2. Git pull로 최신 버전 확보
3. 세션 재시작
4. /memory 명령어로 강제 로드
```

### **Q: 두 Claude가 다른 정보를 가지고 있어요**
```
A:
1. GitHub에서 CLAUDE.md 확인
2. 최신 버전이 맞는지 확인
3. 양쪽에서 모두 최신 버전 pull
4. 세션 재시작
```

### **Q: CLAUDE.md가 너무 길어져요**
```
A:
1. 완료된 오래된 작업은 HISTORY.md로 이동
2. 중요한 것만 CLAUDE.md에 유지
3. 상세 내용은 별도 파일로 분리
4. 주기적으로 정리 (월 1회)
```
